# 产品升级改动文档

## 需求背景

用户需要将处理后的关键词数据导出为 CSV 格式，以便与 AI 大模型进行交互式筛选和分析（如删除品牌词、保留特定关键词等）。CSV 作为纯文本格式，大模型可以直接读取和操作，比 XLSX 更友好。

## 改动清单

### 1. 上传入口：支持 CSV 文件拖入

**文件**: `window.html`
- 文件选择器的 `accept` 属性增加 `.csv`

**文件**: `window.js`
- `handleFileSelect()` / `addFiles()` 中的文件类型验证增加 `.csv` 后缀判断
- `processFileForOcean()` 中根据文件扩展名选择解析方式：
  - `.xlsx` / `.xls` → 现有的 `XLSX.read(data, { type: 'array' })`
  - `.csv` → `XLSX.read(data, { type: 'string' })` 或 `XLSX.read(data, { type: 'array' })` 均可，SheetJS 自动识别格式

**注意**: CSV 和 XLSX 文件可以混合上传，内部数据结构统一为 JSON 数组，后续处理流程无需任何修改。

### 2. 新增"导出 CSV"按钮

**文件**: `window.html`
- 在"下载合并文件"按钮后新增一个 "导出CSV" 按钮
- "下载合并文件" 改名为 "下载合并文件(XLSX)"，明确格式
- 新按钮默认 `disabled`，处理完成后启用

### 3. 导出 CSV 功能

**文件**: `window.js`

新增 `exportCSV()` 方法：
- 将所有 `processedFiles` 的数据合并为**一个 CSV 文件**
- 表头只出现一次（取自第一个文件的表头）
- 所有文件的数据行连续拼接
- **不加**"来源文件名"列
- CSV 列顺序与 XLSX 导出一致：Keyword, Translation, Intent, Volume, Keyword Difficulty, CPC (USD), Kdroi, SERP, Google Trends, Ahrefs Keyword Difficulty Checker
- 数字字段（Kdroi 等）保持数值格式，不强制加引号
- 编码使用 UTF-8 with BOM，确保 Excel 打开中文不乱码
- 文件名格式：`关键词分析结果_YYYY-MM-DD.csv`

### 4. 按钮状态管理

**文件**: `window.js`
- `exportCSV` 按钮与 `downloadBtn` 共用同一状态：处理完成时启用，清空/未处理时禁用
- 在 `startProcessing()` 完成时启用导出 CSV 按钮
- 在 `clearFiles()` 时禁用导出 CSV 按钮

## 不改动的部分

- 现有的 XLSX 导出功能（`downloadMergedFile()`）完全不动
- 关键词海洋处理流程不动
- 翻译、Kdroi 计算、链接生成逻辑不动
- API 配置和性能日志不动
